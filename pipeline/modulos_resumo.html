Módulos
VectorStore
vector_store.py
ABC
Contrato abstracto para todos os stores vectoriais. Define a interface que tanto o ChromaStore como o PostgresStore são obrigados a implementar. Garante que a troca entre backends é uma alteração de uma linha.
add(text, metadata) → str
search(query, filters, n) → list[dict]
remove(doc_id) → None
count() → int
ChromaStore
chroma_store.py
IMPL · agora
Implementação de VectorStore usando ChromaDB. Ranking automático por similaridade cosine, metadados nativos, zero configuração de infraestrutura. Ponto de partida recomendado.
score= 1 − cosine_distance (0→1)
filtroswhere={"type": "ruling"}
deps chromadb sentence-transformers
PostgresStore
postgres_store.py
IMPL · futuro
Implementação de VectorStore usando PostgreSQL + pgvector. Mesma interface que o ChromaStore. Cria automaticamente a tabela e o índice ivfflat na primeira execução. Migração = trocar uma linha de código.
operador<=> cosine (pgvector)
índiceivfflat auto-criado
deps psycopg2 pgvector sentence-transformers
Chunker
chunker.py
ABC
Contrato abstracto para estratégias de divisão de texto. Todas as implementações devolvem a mesma estrutura de dicionário, tornando a troca de estratégia transparente para o resto do pipeline.
split(text) → list[dict]
garantetext · chunk_index · chunk_total
FixedChunker
fixed_chunker.py
IMPL · agora
Divide texto em chunks de tamanho fixo (caracteres) com sobreposição configurável. Sem dependências externas. Ponto de partida — usar eval_rag.py para encontrar os melhores valores de chunk_size e overlap.
chunk_sizedefault 1000 chars
overlapdefault 200 chars (20%)
step= chunk_size − overlap
StructureAwareChunker
structure_aware_chunker.py
IMPL · futuro
Detecta secções do PDF a partir de propriedades tipográficas (tamanho de fonte, boldness) — independente do texto. Subdivide secções longas com FixedChunker internamente. Metadados ricos: section, level, page.
extrasection · section_level · page_start
fallbacktexto simples → FixedChunker
deps pdfplumber FixedChunker
eval_rag
eval_rag.py
AVALIAÇÃO
Avalia combinações de (chunk_size, overlap) contra um conjunto de perguntas com resposta conhecida. Produz Hit Rate e MRR para cada configuração e identifica a melhor automaticamente.
Hit Ratechunk certo nos top-N? (0→1)
MRRMean Reciprocal Rank (1st > 5th)
configs7 combinações testadas por omissão
deps ChromaStore FixedChunker
Embedder
embedder.py
ABC
Contrato abstracto para modelos de embedding. Converte texto em vectores densos. Desacoplado do VectorStore — podes trocar o modelo de embedding sem tocar no store, e vice-versa.
embed(text) → list[float]
dimension→ int   tamanho fixo do vector
model_name→ str   identificador do modelo
SentenceTransformerEmbedder
sentence_transformer_embedder.py
IMPL · agora
Modelo local via sentence-transformers. Download automático na primeira execução, depois funciona offline. Modelo recomendado para o AIAA: paraphrase-multilingual-MiniLM-L12-v2 (384-dim, suporta português).
defaultparaphrase-multilingual-MiniLM-L12-v2
dim384  (ou conforme o modelo escolhido)
deps sentence-transformers
OllamaEmbedder
ollama_embedder.py
IMPL · futuro
Modelo open-source servido localmente pelo Ollama. Sem API keys, sem custos, sem internet. Alinha com o requisito da proposta de usar LLMs open-source. Dimensão resolvida automaticamente na primeira chamada.
defaultnomic-embed-text (768-dim)
alternativamxbai-embed-large (1024-dim)
deps Ollama (serviço local)
PDFLoader
pdf_loader.py
ABC
Entrada do pipeline. Lê um PDF e produz texto pronto a passar a qualquer Chunker. Duas implementações: TextLoader para texto simples (usa com FixedChunker) e PlumberLoader para inspecção tipográfica (usa com StructureAwareChunker).
load() → str   texto completo do PDF
metadata() → dict   filename · pages · source
font_sizes() → dict   frequência por tamanho (PlumberLoader)
impls TextLoader PlumberLoader deps pdfplumber
example_usage
example_usage.py
GLUE
Script de demonstração do pipeline completo. Mostra como indexar documentos e pesquisar. Serve também como template para o código de aplicação real que virá a seguir.
index_document(text, metadata, store, chunker)
deps ChromaStore FixedChunker StructureAwareChunker
LLM
llm.py
ABC
Contrato abstracto para modelos de linguagem. Recebe um prompt completo e devolve texto gerado. Desacoplado do PromptBuilder — a construção do prompt e a geração são responsabilidades separadas.
generate(prompt) → str
model_name→ str
OllamaLLM
ollama_llm.py
IMPL · produção
LLM open-source servido localmente pelo Ollama. Sem API keys, sem custos, dados nunca saem da máquina. Modelos recomendados para português jurídico: mistral, llama3, qwen2.
temperature0.0 recomendado (respostas deterministas)
defaultmistral · timeout=120s
deps Ollama (serviço local)
EchoLLM
echo_llm.py
IMPL · testes
Stub de testes que devolve o prompt sem gerar texto. Permite testar o Retriever e o PromptBuilder sem Ollama ou GPU. Output prefixado com [ECHO] para ser óbvio nos logs.
generate(prompt) → "[ECHO]\n" + prompt
PromptBuilder
prompt_builder.py
ABC
Contrato abstracto para construção de prompts. Recebe a pergunta e os chunks recuperados e devolve um prompt completo. Separado do Retriever porque o formato do prompt tem impacto directo na qualidade e requer iteração frequente.
build(question, chunks) → str
BasicPromptBuilder
basic_prompt_builder.py
IMPL · baseline
Template simples e agnóstico à língua. Numera os chunks como contexto e adiciona a pergunta. Usar como baseline para comparar com o LegalPromptBuilder na avaliação de qualidade.
max_chunksdefault 5
LegalPromptBuilder
legal_prompt_builder.py
IMPL · produção
Template especializado para contexto jurídico português. Instrui o LLM a responder em português formal, citar fontes por índice, e declarar explicitamente quando o contexto é insuficiente. Nunca inventar referências ou artigos de lei.
min_score0.3 — filtra chunks pouco relevantes
max_chunksdefault 5
Retriever
retriever.py
ORQUESTRADOR
Orquestra o fluxo completo de interrogação. Recebe os quatro componentes injectados e expõe um único método ask() que devolve resposta + fontes usadas + prompt enviado ao LLM.
ask(question) → RetrievalResult
RetrievalResultanswer · sources · prompt
injected Embedder VectorStore PromptBuilder LLM
